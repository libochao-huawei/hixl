1. 概述：UBG 是什么

UBG（Unified Bus over Gige）是昇腾 A5（Ascend950）平台上的一种 scale-out（跨超节点）单边通信协议， 与已有的 UBoE 并列。两者都基于 EID（Endpoint Identifier）寻址，通过 hcomm 底层库完成建链和数据传输。

UBG vs UBoE vs RoCE

协议寻址方式EID 标记Host 内存
UBGEID（16 字节）byte[7] & 0xC0 == 0x80需 HostRegister 映射
UBoEEID / IPbyte[7] & 0xC0 == 0xC0需 HostRegister 映射
RoCEIP 地址—直接注册

适用平台

平台支持的 scale-out 协议
A2 (910B)RoCE
A3 (910)RoCE
A5 (950)UBG / UBoE / RoCE

2. 核心概念与数据结构

2.1 EID 编码（UBG vs UBoE 区分）

EID 是 16 字节原始地址，HIXL 通过第 7 字节（raw[7]）的高 2 位区分协议类型：

// endpoint_generator.cc:44-46
constexpr size_t kUbgEidMarkerByteIndex = 7U;
constexpr uint8_t kUbgEidMarkerMask    = 0xC0U;
constexpr uint8_t kUbgEidMarkerValue   = 0x80U;  // 高2位=10 → UBG; 高2位=11(0xC0) → UBoE

// 判定函数
bool IsUbgEid(const DcmiUrmaEidInfo &eid_info) {
    return (eid_info.eid.raw[kUbgEidMarkerByteIndex] & kUbgEidMarkerMask) == kUbgEidMarkerValue;
}
十六进制示例：UBG EID = 0000000000ff0a80...（byte[7]=0x80）；UBoE EID = 0000000000ff0ac0...（byte[7]=0xC0）。HIXL 将 16 字节转为 32 字符 hex 字符串作为 comm_id。

2.2 DSMI InterconType（互联类型枚举）

DSMI 接口 GetInterconType 返回硬件的 scale-out 互联类型，驱动据此自动选择生成哪种 endpoint：

值含义HIXL 生成判定函数
0UBoE over Switch（David→UBG→5808）UBoE endpointIsUboeInterconType
1RoCE over NPU（1825，驱动默认）不生成，回退旧路径IsRoceInterconType
2UBoE over NPU（David UBoE）UBoE endpointIsUboeInterconType
3RoCE over CPU（Host NIC）不生成IsRoceInterconType
4UBG over NPU（David UBG）UBG endpointIsUbgInterconType

2.3 ProtocolLock（协议锁定）

当用户通过 protocol_desc 显式指定协议时，HIXL 设置 ProtocolLock，在匹配阶段强制仅匹配该协议：

2.4 net_instance_id 与超节点判定

net_instance_id = to_string(super_pod_id)，通过 ACL 接口获取。匹配器用它判断两台机器是否在同一超节点内：

// endpoint_matcher.cc:76
bool IsCrossSuperNode(local, remote) {
    return local[0].net_instance_id != remote[0].net_instance_id;
}
同超节点 → 优先 HCCS/UB；跨超节点 → 优先 UBoE > UBG > RoCE。

3.2 Endpoint 自动生成的三条路径

AutoGenEndpointList 中 kV5 分支按优先级依次尝试三条路径，先到先得（互斥）：

路径 1：DSMI InterconType 自动生成（首选）

DSMI 接口可用时，根据硬件互联类型自动决定生成 UBG 或 UBoE endpoint。

// endpoint_generator.cc:204-234
GenerateKv5EndpointByInterconType(logic_dev_id, phy_dev_id, endpoint_list) {
    if (!DsmiProxy::IsInterconTypeSupported()) {
        return SUCCESS;  // list 为空，回退路径 2
    }
    DsmiProxy::GetInterconType(logic_dev_id, intercon_type);
    if (IsUbgInterconType(intercon_type))   // == 4

    else if (IsUboeInterconType(intercon_type))  // == 0 或 2

}

3.3 UBG EID 获取详解（GenDefaultUbgEndpointConfig）

这是 UBG 最核心的硬件交互：从 DCMI 获取 URMA 设备的 EID 列表，筛选出 UBG EID。

// endpoint_generator.cc:132-161
Status GetUbgEidFromDcmi(int32_t phy_dev_id, std::string &eid) {
    // 1. phy_id → logic_id
    DcmiProxy::GetLogicIdFromPhyId(phy_dev_id, &logic_id);           DCMI

    // 2. 获取 URMA 设备数量
    DcmiProxy::GetUrmaDeviceCnt(logic_id, &dev_cnt);                  DCMI

    // 3. 遍历每个 URMA 设备的 EID 列表
    for (dev_index = 0; dev_index < dev_cnt; ++dev_index) {
        DcmiProxy::GetEidList(logic_id, dev_index, eid_list[32], &eid_cnt);  DCMI

        for (eid_index = 0; eid_index < eid_cnt; ++eid_index) {
            // 跳过全零 EID
            if (all_of(eid_list[eid_index].eid.raw == 0)) continue;

            // 检查标记字节：raw[7] & 0xC0 == 0x80 → UBG
            if (IsUbgEid(eid_list[eid_index])) {
                eid = ConvertEidToString(eid_list[eid_index].eid.raw, 16);
                return SUCCESS;  // 16字节 → 32字符hex
            }
        }
    }
    return FAILED;  // 未找到 UBG EID
}

4.2 UBG Host 内存的特殊处理

UBG 与 UBoE 一样属于 IsHostRegisterMappedProtocol，Host 内存不能直接注册到 hcomm，需要先映射为 device 地址：

// endpoint.cc:82-95 — RegisterMem 中的 Host 映射逻辑
if (IsHostRegisterMappedProtocol(endpoint_.protocol) && mem.type == COMM_MEM_TYPE_HOST) {
    // UBG 或 UBoE + Host 内存 → 先映射
    HostRegisterProxy::RegisterByDev(devPhyId, mem.addr, mem.size, &registered_dev_mem);

    // 用映射后的 device 地址注册到 hcomm
    reg_mem.type = COMM_MEM_TYPE_DEVICE;
    reg_mem.addr = registered_dev_mem;  // 映射后的地址
    HcommProxy::MemReg(handle_, mem_tag, &reg_mem, &mem_handle);
} else {
    // Device 内存直接注册
    HcommProxy::MemReg(handle_, mem_tag, &mem, &mem_handle);
}
关键点：传输时也需将用户传入的 host 地址转换为映射后的 device 地址。HixlCSClient::ConvertHostMappedDescs 负责这个转换（hixl_cs_client.cc:1028），否则数据会写到错误地址。

7. Endpoint 匹配与协议选取

Connect 阶段调用 EndpointMatcher::MatchEndpoints 决定用哪对 endpoint 建链。这是 scale-out 自动选取的核心。

7.1 匹配流程

MatchEndpoints(local, remote, pairs, handler_type, protocol_lock) // endpoint_matcher.cc:182
// 1. 判断场景
cross_supernode = IsCrossSuperNode(local, remote) // net_instance_id 比较
scenario = DetermineScenario(cross_supernode):
if (IsForceRoceOnly()) → kForceRoce // HCCL_INTRA_ROCE_ENABLE=1
else if (cross_supernode) → kCrossSuperNode
else → kSameSuperNode
// 2. 构建优先级表
priority = BuildPriority(protocol_lock, cross_supernode)
// 3. 按优先级依次尝试匹配
for (entry in priority):
if (IsUbProtocol(entry.protocol))
TryMatchUbAll() // UB 多链路组匹配
else
MatchDirect() // 单链路直接匹配（检查 protocol + placement）
if matched → return SUCCESS
return PARAM_INVALID

7.2 优先级表

场景优先级（从高到低）
kForceRoceRoCE:device → RoCE:host
kCrossSuperNodeUBoE:device → UBG:device → RoCE:device → RoCE:host
kSameSuperNodeUB(group) → HCCS:device → UBoE:device → UBG:device → RoCE:device → RoCE:host

7.3 ProtocolLock 过滤

当 protocol_lock = kUbg 时，优先级表中仅保留 UBG 条目，强制走 UBG：

// endpoint_matcher.cc:177-179
if (scenario != kForceRoce && protocol_lock != kNone) {
    const char *locked = (protocol_lock == kUbg) ? kProtocolUbg : kProtocolUboe;
    priority.erase(remove_if(priority, [locked](e) { return e.protocol != locked; }));
    // kForceRoce 场景下 protocol_lock 被忽略 — HCCL_INTRA_ROCE_ENABLE=1 优先级最高
}

7.4 MatchDirect — placement 一致性校验

// endpoint_matcher.cc:91-106
Status MatchDirect(local, remote, entry, pairs) {
    // 同时校验 protocol 和 placement（nullptr 表示不限，仅 UB 条目）
    auto is_matched = [&entry](const EndpointConfig &ep) {
        return ep.protocol == entry.protocol &&
               (entry.placement == nullptr || ep.placement == entry.placement);
    };
    auto local_it  = find_if(local.begin(),  local.end(),  is_matched);
    auto remote_it = find_if(remote.begin(), remote.end(), is_matched);
    if (local_it != local.end() && remote_it != remote.end()) {
        pairs.push_back({*local_it, *remote_it, entry.comm_type});
        return SUCCESS;
    }
    return FAILED;
}
