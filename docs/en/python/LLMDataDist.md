# LLMDataDist

## Product Support

| Product | Supported |
| --- | --- |
| Ascend 950PR/Ascend 950DT | Yes |
| Atlas A3 training series products/Atlas A3 inference series products | Yes |
| Atlas A2 training series products/Atlas A2 inference series products | Yes |

Note: For Atlas A2 training series products/Atlas A2 inference series products, only Atlas 800I A2 inference server/A200I A2 Box heterogeneous component is supported. For Ascend 950PR/Ascend 950DT, link, unlink, and query_register_mem_status are not supported.

## LLMDataDist Constructor

**Function**

Constructs LLMDataDist.

**Prototype**

```
__init__(role: LLMRole, cluster_id: int)
```

**Parameters**

| Parameter Name | Data Type | Value Description |
| --- | --- | --- |
| role | LLMRole | Cluster role. This parameter is used only to identify the current role and has no impact on the transfer process. Values are as follows.<br><br> - LLMRole.DECODER: incremental cluster<br> - LLMRole.PROMPT: full cluster |
| cluster_id | int | Cluster ID. LLMDataDist identifier. It must be unique within the scope of all participants in link establishment. |

**Example**

```
from llm_datadist import LLMDataDist, LLMRole
llm_datadist = LLMDataDist(LLMRole.DECODER, 0)
```

**Return Value**

Returns an LLMDataDist instance in normal cases.

Parameter errors may throw TypeError or ValueError.

**Constraints**

None

## Init

**Function**

Initializes LLMDataDist. The options of init must configure CacheManager mode, that is, enable\_cache\_manager is set to True or local\_comm\_res is specified.

In this mode, Decode and Prompt can pull Cache from each other.

**Prototype**

```
init(options: Dict[str, str])
```

**Parameters**

| Parameter Name | Data Type | Value Description |
| --- | --- | --- |
| options | Dict[str, str] | Configuration items.<br>The passed options can be generated by LLMConfig.<br><br> - device_id is required.<br> - enable_cache_manager is True or local_comm_res is set. |

**Example**

```
from llm_datadist import LLMDataDist, LLMRole, LLMConfig
llm_datadist = LLMDataDist(LLMRole.PROMPT, 0)
llm_config = LLMConfig()
llm_config.enable_cache_manager = True
llm_config.device_id = 0
engine_options = llm_config.generate_options()
llm_datadist.init(engine_options)
```

**Return Value**

In normal casesNo return value.

Throws LLMException in abnormal cases.

Parameter errors may throw TypeError or ValueError.

**Constraints**

After initialization succeeds, finalize must be called before the system exits.

## Link\_clusters

**Function**

One-sided link establishment is initiated by the Client. Whether the end is Client or Server is unrelated to the prompt or decoder role. Setting listen\_ip\_info identifies the listening port, which means the end is the Server.

**Prototype**

```
link_clusters(clusters: Union[List[LLMClusterInfo], Tuple[LLMClusterInfo]], timeout=3000)
```

**Parameters**

| Parameter Name | Data Type | Value Description |
| --- | --- | --- |
| clusters | Union[List[LLMClusterInfo], Tuple[LLMClusterInfo]] | Cluster list. |
| timeout | int | Timeout. Unit: ms. Default timeout: 3000 ms. |

**Example**

See [Example Execution](../../../examples/python/README_en.md).

**Return Value**

In normal cases, returns a tuple of two values. The first value is the API return value of type LLMStatusCode. The second value is a list of link establishment results for each cluster, of type LLMStatusCode.

Parameter errors may throw TypeError or ValueError.

**Constraints**

- The link establishment requirements are as follows. Too many links may cause memory OOM and KV Cache transfer performance risks.
  - The maximum number of allowed communications is 512.

- It is recommended to set the timeout to more than 200 ms. If TLS is enabled, it is recommended to set the timeout to more than 2000 ms. Use the following command to query TLS status: 

  ```
  hccn_tool [-i %d] -tls -g [host]
  ```

- Register all memory before calling this API. Otherwise, memory registered after link establishment does not support remote access.
- In container scenarios, if local\_comm\_res is not configured or is empty, map the "/etc/hccn.conf" file in the container or ensure that hccn_tool exists in the default path "/usr/local/Ascend/driver/tools", If neither condition is met, configure the path where hccn_tool is located in PATH.The configuration example is as follows, hccn_tool_install_path indicates the path where hccn_tool is located.

  ```
  export PATH=$PATH:{hccn_tool_install_path}
  ```

## Unlink\_clusters

**Function**

One-sided unlink can be initiated by the Client to notify the Server to unlink. Alternatively, both the Client and Server can initiate forced unlink, which clears only the local link.

**Prototype**

```
unlink_clusters(clusters: Union[List[LLMClusterInfo], Tuple[LLMClusterInfo]], timeout=3000, force=False)
```

**Parameters**

| Parameter Name | Data Type | Value Description |
| --- | --- | --- |
| clusters | Union[List[LLMClusterInfo], Tuple[LLMClusterInfo]] | Cluster list. |
| timeout | int |Timeout. Unit: ms. Default timeout: 3000 ms. |
| force | bool | Specifies whether to force unlink. The default value is False. True indicates forced unlink.<br><br> - Forced unlink removes only the local link. Both ends must call it.<br> - Non-forced unlink is initiated by the Client. If no fault occurs, links on both ends are removed. If unlink fails due to a fault, unlink must also be initiated on the Server end.<br> - If no fault occurs, links on both ends are removed.<br> - If unlink fails due to a fault, unlink must also be initiated on the Server end. |

**Example**

See [Example Execution](../../../examples/python/README_en.md).

**Return Value**

In normal cases, returns a tuple of two values. The first value is the API return value of type LLMStatusCode. The second value is a list of link establishment results for each cluster, of type LLMStatusCode.

Parameter errors may throw TypeError or ValueError.

**Constraints**

None

## Switch\_role

**Function**

Switches the role of the current LLMDataDist. switch\_options can also be configured to switch between Client and Server.

**Prototype**

```
switch_role(self, role: LLMRole, switch_options: Optional[Dict[str, str]] = None)
```

**Parameters**

| Parameter Name | Data Type | Value Description |
| --- | --- | --- |
| role | LLMRole | Target role to switch to. |
| switch_options | options: Dict[str, str] | Role switching options.<br>Optional parameter. The default value is None.<br>If used as Server, configure the listening Host IP and port through listen_ip_info, such as "192.168.1.1:26000". If it is not set, the end is used as Client.<br>Switching from Server to Client closes the previously listened port. To switch only the listening port of the Server, use options to set another listening port, which closes the previous listening port.<br>When this configuration item specifies the hixl transfer backend, changing the listening port through switch\_role is not supported. |

**Example**

See [Example Execution](../../../examples/python/README.md).

**Return Value**

- In normal casesNo return value.
- If the input data type is incorrect, TypeError or ValueError is thrown.
- If residual link resources exist during switch\_role, LLMException is thrown, and status\_code is LLM\_EXIST\_LINK.

**Constraints**

None

## Link

**Function**

Two-sided link establishment establishes a link by creating a communication domain. The easier-to-use one-sided link\_clusters API is recommended because it does not require ranktable information.

To use RDMA NIC communication on a single machine, set the HCCL\_INTRA\_ROCE\_ENABLE=1 environment variable.

**Prototype**

```
link(comm_name: str, cluster_rank_info: Dict[int, int], rank_table: str) -> int
```

**Parameters**

| Parameter Name | Data Type | Value Description |
| --- | --- | --- |
| comm_name | str | Communication domain name.<br>Value range: fewer than 128 characters. |
| cluster_rank_info | Dict[int, int] | Mapping from cluster ID to rank ID.<br>Example: {1: 0, 2: 1}. |
| rank_table | str | Developers can use this parameter to configure NPU resource information participating in collective communication.

For details about ranktable in the preceding table, see [HCCL Collective Communication Library User Guide](https://www.hiascend.com/document/detail/en/canncommercial/850/API/ascendcopapi/atlasascendc_api_07_0869.html).<br>The configuration example of rank_table in the preceding table is as follows:

```
{
  "server_count": "2",
  "server_list": [
    {
      "device": [
        {
          "device_id": "0",
          "device_ip": "x.x.x.x",
          "rank_id": "0"
        },
      ],
      "server_id": "xxxx"
    },{
      "device": [
        {
          "device_id": "0",
          "device_ip": "x.x.x.x",
          "rank_id": "1"
        },
      ],
      "server_id": "xxxx"
    }
  ],
  "status": "completed",
  "version": "1.0"
}
```

**Example**

See [Example Execution](../../../examples/python/README_en.md).

**Return Value**

In normal cases, returns the ID that identifies the communication domain.

Parameter errors may throw TypeError or ValueError.

**Constraints**

- All nodes in the communication domain need to initiate the operation at the same time.
- The maximum number of nodes in a communication domain is 4.
- It is recommended that the number of communication domains does not exceed 16. The maximum is 512. Too many links may cause memory OOM and cache transfer performance degradation risks.
- The underlying link exchanges memory descriptors inside the communication domain, For the cache used for transfer, configure the memory pool or register memory before link.
- Calling through forked child processes is not supported.
- A maximum of 16 links can be established concurrently. If there are more than 16 links, the underlying layer queues them.
- Ensure that no circular dependency occurs when establishing links for multiple communication domains.
- This API is not supported on Ascend 950PR/Ascend 950DT.
