## HIXL在昇腾A2芯片上部分场景实测性能数据

本页展示 HIXL 在 **昇腾 A2**（Atlas 800I A2 / A200I A2 Box）上的通信性能数据。

**测试方式**：在同一台机器上运行 `hixl_comm_bench` 的 target 和 initiator，测量不同 block size（16K～2M）下的有效带宽。

**约束**：A2 上 HCCS 传输仅支持 Device↔Device（D2D）方向；其他方向（D2H、H2H、H2D）只能走 ROCE（RDMA）传输。

### 如何读懂表格

- **Block** 列：每次传输的数据块大小（16K = 16 KiB，1M = 1 MiB）。
- **方向+传输** 列：例如 `D2rD HCCS` 表示从本地 Device 写往远程 Device（HCCS 传输），`rH2D ROCE` 表示从远程 Host 读回本地 Device（ROCE 传输）。
- 数值为有效带宽（GB/s），**——** 表示该组合不支持或尚未测试。
- 所有数据来自单机 1:1（一个 initiator + 一个 target）测试。

### 方向命名速查

方向名格式为 `源 → 远程目标`，其中 **D**=Device、**H**=Host、**r**=remote。

| 方向 | 含义 | Initiator | Target | 操作 |
| :--- | :--- | :---: | :---: | :---: |
| **D2rD** | D → rD (write D2D) | Device | Device | write |
| **D2rH** | D → rH (write D2H) | Device | Host | write |
| **H2rH** | H → rH (write H2H) | Host | Host | write |
| **H2rD** | H → rD (write H2D) | Host | Device | write |
| **rD2D** | rD → D (read D2D) | Device | Device | read |
| **rH2D** | rH → D (read D2H) | Device | Host | read |
| **rH2H** | rH → H (read H2H) | Host | Host | read |
| **rD2H** | rD → H (read H2D) | Host | Device | read |

## HIXL在昇腾A3芯片上部分场景实测性能数据

**约束**：A3 上 HCCS 传输不支持 Host 内存作为远端 Cache；D2H/H2H 方向的 HCCS 数据因此缺失。ROCE 和 FabricMem 支持全部方向。

### 方向命名速查

方向名格式为 `源 → 远程目标`，其中 **D**=Device、**H**=Host、**r**=remote。

| **D2rH** | D → rH (write D2H) | Device | Host | write |
